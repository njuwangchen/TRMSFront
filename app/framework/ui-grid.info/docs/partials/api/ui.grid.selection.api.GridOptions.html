GridOptions (api in module ui.grid.selection )

Description

GridOptions for selection feature, these are available to be set using the ui-grid gridOptions

Properties

  • enableFooterTotalSelected

    Shows the total number of selected items in footer if true.
    Defaults to true.
    GridOptions.showFooter must also be set to true.

  • enableRowHeaderSelection

    Enable a row header to be used for selection
    Defaults to true

  • enableRowSelection

    Enable row selection for entire grid.
    Defaults to true

  • enableSelectAll

    Enable the select all checkbox at the top of the selectionRowHeader
    Defaults to true

  • enableSelectionBatchEvent

    If selected rows are changed in bulk, either via the API or via the selectAll checkbox, then a separate event is fired. Setting this option to false will cause the rowSelectionChanged event to be called multiple times instead
    Defaults to true

  • isRowSelectable

    Makes it possible to specify a method that evaluates for each and sets its "enableSelection" property.

  • modifierKeysToMultiSelect

    Enable multiple row selection only when using the ctrlKey or shiftKey. Requires multiSelect to be true.
    Defaults to false

  • multiSelect

    Enable multiple row selection for entire grid
    Defaults to true

  • noUnselect

    Prevent a row from being unselected. Works in conjunction with multiselect = false and gridApi.selection.selectRow() to allow you to create a single selection only grid - a row is always selected, you can only select different rows, you can't unselect the row.
    Defaults to false

  • selectionRowHeaderWidth

    can be used to set a custom width for the row header selection column
    Defaults to 30px
